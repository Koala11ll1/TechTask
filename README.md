# Ticket Notification System (.NET 8 Web API)

Це навчальний проєкт системи обробки тікетів та розсилки сповіщень через різні канали (Email, SMS, Push). Проєкт побудований з дотриманням принципів **SOLID** та архітектурних патернів ASP.NET Core.

## 🚀 Основні можливості

* **Керування тікетами**: Створення тікетів та автоматична генерація черги сповіщень.
* **Мультиканальність**: Підтримка відправки через Email, SMS та Push-нотифікації.
* **Retry Logic**: Автоматична система повторних спроб (до 3 разів) у разі помилки відправки.
* **Ідемпотентність**: Захист від повторної відправки вже доставлених сповіщень.
* **In-Memory Storage**: Використання Singleton-сховища для роботи без зовнішньої БД (зручно для тестування).

## 🛠 Технологічний стек

* **Framework**: .NET 8.0 (ASP.NET Core Web API)
* **DI Container**: Вбудований Microsoft.Extensions.DependencyInjection
* **Testing**: xUnit, Moq
* **Documentation**: Swagger / OpenAPI

## 🏗 Архітектура та Патерни

Проєкт демонструє практичне застосування таких концепцій:

1.  **Single Responsibility (SRP)**: Логіка відправки винесена з контролера в `NotificationService`.
2.  **Open/Closed (OCP)**: Додавання нового каналу зв'язку (наприклад, Telegram) не потребує зміни існуючого коду сервісу.
3.  **Dependency Inversion (DIP)**: Використання інтерфейсів (`ITicketStorage`, `INotificationSender`) для гнучкої заміни реалізацій.
4.  **Strategy Pattern**: Динамічний вибір потрібного сендера на основі типу каналу сповіщення.



## 📋 Як запустити

1. Клонуйте репозиторій.
2. Відкрийте рішення (`.sln`) у Visual Studio 2022.
3. Натисніть `F5` для запуску проєкту.
4. Перейдіть за адресою `https://localhost:XXXX/swagger` для доступу до API.

## 🧪 Тестування

Проєкт містить 6 Unit-тестів, що покривають критичну бізнес-логіку:
* Перевірка створення 3-х каналів сповіщення для кожного тікета.
* Перевірка ліміту спроб відправки (`MaxRetryAttempts`).
* Тестування ідемпотентності (захист від дублікатів).
* Обробка помилок та запис `LastError`.

Щоб запустити тести, відкрийте **Test Explorer** у Visual Studio та натисніть **Run All**.

## 📖 Ендпоінти API

* `POST /api/tickets` — Створення нового тікета.
* `GET /api/tickets/{id}` — Отримання даних тікета та статусів його сповіщень.
* `POST /api/tickets/{id}/notify` — Запуск процесу розсилки сповіщень для конкретного тікета.
